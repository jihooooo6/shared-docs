핵심 성과:

2. Skills 시스템

생성된 Skills

~/.claude/skills/
├── github-share/SKILL.md - GitHub 문서 공유
├── config-sync/SKILL.md - 설정 동기화 (안전장치 포함)
├── query-request/SKILL.md - 쿼리 요청
└── plan-mode/SKILL.md - 플랜/계획

Skills vs CLAUDE.md

구분 CLAUDE.md Skills
로드 시점 매 턴마다 항상 필요할 때만
컨텍스트 항상 소모 해당 작업 시에만
용도 항상 적용되는 핵심 규칙 특정 작업 상세 절차
주의: Skills가 100% 자동 적용되지는 않음 (AI 한계). 필요시 "저장해줘", "파일 만들어줘" 등 명시적 요청 가능.

3. CLAUDE.md 최종 구조

섹션 내용
1. 답변 규칙 질문 반복, Skills 사용 목록 표시, 이모지 금지
2. 코딩 스타일 javadoc, 쿼리 주석
3. 폴더 접근 규칙 .cursor/.vscode 무시, IntelliJ 사용
4. 환경 자동 인식 회사(JJI) / 집(foeve) 구분
5. 작업 파일 저장 my-answer 폴더, Q/P/S/R 접두사, 쿼리 요청 규칙
6. 파일 수정 규칙 즉시 수정, 재기동 명시, 변경 파일 목록
7. 테스트 및 빌드 사용자 직접 수행
8. Skills 목록 4개 스킬 트리거 정보
9. 최종 원칙 프로젝트별 설정 우선

추가된 규칙

### 쿼리 요청 규칙 * 사용자가 "저장해줘"라고 하면 해당 내용을 Q_ 파일로 저장한다. * 작업 중 DB 데이터가 필요하면 Q_요청.txt 파일을 생성하고 사용자에게 실행 요청한다. * 사용자가 Q_답변.txt에 결과를 입력하면 해당 파일을 읽고 작업을 계속한다.

4. 설정 동기화 체계

GitHub 저장소

저장소 용도 공개여부
claude-config CLAUDE.md, settings.json, skills/ 동기화 Private
shared-docs 고객/외부 공유 문서 Public
work-docs 내부 업무 문서 Private
public-docs 포트폴리오/공개용 Public

동기화 명령어

명령 방향 동작
"설정 업로드해줘" 로컬 → GitHub 내 변경사항 저장
"설정 동기화해줘" GitHub → 로컬 GitHub 내용으로 적용
주의: 로컬에서 수정 후 "동기화해줘" 하면 변경사항 날아감! 반드시 "업로드" 먼저!

안전장치

동기화 전 로컬 변경사항 체크 → 변경사항 있으면 "업로드 먼저 할까요?" 확인
